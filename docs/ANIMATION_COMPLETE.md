## 🔑 Key Features

✅ **Scroll-Triggered**
- Animates when 20% of element enters viewport
- Only once per page load
- No performance impact

✅ **Accessibility**
- Auto-detects `prefers-reduced-motion`
- Content appears instantly if motion disabled
- WCAG 2.1 AA compliant

✅ **Performance**
- GPU-accelerated (transform + opacity only)
- 60 FPS smooth animations
- Lazy-loaded library (+27 kB only when used)
- No layout shifts (no CLS)

✅ **Easy to Use**
- Just wrap with `<FadeUpSection>`
- No prop configuration needed
- Intelligent defaults

✅ **Responsive**
- Works on all devices
- Mobile-friendly
- Touch-friendly

```jsx
    </ScaleInCard>
  </StaggerItem>

// Do this

```

### Tip 2: Test Accessibility
Always test with `prefers-reduced-motion` enabled to ensure content is still visible and readable.

### Tip 3: Monitor Performance
After deploying, check Core Web Vitals to ensure animations don't impact real-world performance.

### Tip 4: Ask Users
Gather feedback on animation timing and feel. Every audience is different.

### Tip 5: Iterate Slowly
Add animations section by section, not all at once. This helps identify any issues.
